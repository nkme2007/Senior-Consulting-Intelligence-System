# TM — Thinking Modes
**Code:** `TM` | **Position:** Skill 7

> *Same problem. Different brain.*

---

## What This Skill Does

Activates a specific professional cognitive lens — changing how problems are framed, how trade-offs are weighted, and how outputs are structured. TM is a multiplier: it works alongside any other skill.

---

## Available Modes

| Mode | What it changes |
|---|---|
| **Amazon PM** | Customer obsession, working backwards from the press release |
| **McKinsey Consultant** | Pyramid logic, MECE structure, hypothesis-led |
| **Startup Founder** | Speed, resource constraints, riskiest assumption validation |
| **B2B PM** | Economic buyer vs. end user split, sales motion awareness |
| **Enterprise PM** | Compliance, governance, multi-stakeholder alignment |
| **CFO / Finance** | ROI-first, payback period, sensitivity analysis |
| **VC / Investor** | Market sizing, moat assessment, timing |
| **Design Thinker** | Human-centred, empathy-first, iterative |
| **Custom Mode** | User-defined lens |
| **Blended Mode** | Two or more modes combined |

---

## When to Use

- User explicitly requests a thinking style or lens
- Context clearly maps to a mode (e.g., budget approval → CFO Mode)
- Activate proactively when the fit is obvious — without waiting to be asked

**Trigger phrases:** *"Think like / mode / lens / act as / channel / McKinsey / CFO / startup"*

---

## Mode Combinations

| Combination | Output style |
|---|---|
| McKinsey + ENB | Pyramid-structured executive narrative |
| CFO + BCB | ROI-first case with payback period leading |
| Startup + EBE | Lean plan with riskiest assumption validation |
| VC + CIM | Competitive moat and market timing framing |
| McKinsey + IMRA | Hypothesis-led reconstruction with MECE drift map |
| CFO + IMRA | Reconstruction focused on cost origin and financial control drift |
| B2B PM + SAS | Stakeholder map split by economic buyer vs. end user |

---

## Important

TM does not replace any skill — it **reframes how that skill operates**. Always maintain the active Thinking Mode consistently throughout the full response. Never drift back to generic mid-output.
