# IMRA — Institutional Memory Reconstruction Agent
**Code:** `IMRA` | **Position:** Skill 10

> *Recover what the system forgot it knows.*

---

## What This Skill Does

Reconstructs why an organisation, process, or policy works the way it does — especially when documentation is missing, messy, or contradictory. Separates what was designed intentionally from what accumulated by accident.

---

## When to Use

- You're inheriting a system, team, or function and need to understand it before changing it
- Documentation is missing, contradictory, or outdated
- Everyone works around a process but no one knows why it exists
- You're leading M&A integration, org transformation, or a governance review

**Trigger phrases:** *"Why do we do it this way / institutional memory / system audit / what broke when we tried changing this / we inherited this / how did we get here"*

---

## Two Operating Modes

**Standalone Mode** — Full reconstruction report delivered as a complete output.

**Embedded Mode** — Modular blocks with handoff flags, designed to feed into other skills (DIE, SAS, BCB, CIM, EBE, TM, MKA, ENB).

---

## Output

- **True optimisation target** — what the system is actually trying to achieve vs. what it says
- **Original design intent (reconstructed)** — why it was built this way, even if no one remembers
- **Drift analysis map** — how the system evolved from intent to current state
- **Sacred vs. accidental rules inventory** — what must stay vs. what can change
- **Change-safe lever map with resistance profile** — where to intervene, and what resistance each lever will trigger

---

## Pairs Well With

| Pair | Why |
|---|---|
| IMRA → DIE | Reconstruct reality, then make the informed decision |
| IMRA → EBE | Understand the system, then plan the change |
| IMRA → SAS | Know the history, then simulate stakeholder resistance |
| IMRA → ENB | Reconstruct, then communicate the change |
| IMRA → MKA | Understand what was measured before, redefine success |
| TM + IMRA | McKinsey Mode = MECE hypothesis-led; CFO Mode = cost and control drift focus |
